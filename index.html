git config --global user.name - Команда для добавления глобального имени пользователя

git config --global user.email - Команда для добавления глобального емейл пользователя

git init - Эта команда создаёт в текущей директории новую поддиректорию с именем .git, содержащую все необходимые файлы репозитория — структуру Git-репозитория.

git status - Отображает пути, которые имеют различия между индексным файлом и текущей фиксацией HEAD, пути, которые имеют различия между рабочим деревом и индексным файлом, а также пути в рабочем дереве, которые не отслеживаются Git (и не игнорируются gitignore )

git add - Эта команда нужна чтобы начать отслеживать (добавить под версионный контроль) новый файл.

git commit - Записывает изменения в репозиторий.

git log - Показать журналы фиксации.

git checkout - Переключение ветвей или восстановление файлов рабочего дерева

git branch - Список, создание или удаление веток

git push - Обновляет удаленные ссылки, используя локальные ссылки, при отправке объектов, необходимых для завершения данных ссылок.

git pull - Включает изменения из удаленного репозитория в текущую ветку. В режиме по умолчанию git pullэто сокращенное обозначение, за git fetchкоторым следует git merge FETCH_HEAD

git merge - Объединяет две или более историй разработки вместе

git clone - Клонирует репозиторий в новый каталог

gitignore служит для указания в нём файлов и папок, которые необходимо скрыть от системы контроля версий git. Как правило, скрывают файлы или папки, которые автоматически генерируют своё содержимое, либо имеют конфигурационные параметры, которые могут различаться у тех, кто совместно работает над проектом.

Advanced

git revert - Отменяет последний комит

git reset - Сбросить текущую HEAD в указанное состояние

git rebase - Команда предназначена, чтобы объединить изменения, сделанные в одной ветке, с другой веткой.

git cherry-pick - Применить изменения, внесенные некоторыми существующими коммитами

git stash - Используйте, если вы хотите записать текущее состояние рабочего каталога и индекса, но хотите вернуться в чистый рабочий каталог. Команда сохраняет ваши локальные изменения и возвращает рабочий каталог в соответствие с HEAD фиксацией.
